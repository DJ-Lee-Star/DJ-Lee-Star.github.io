# 기업여신 AI 시연 컬러정책 및 단말 UI 원칙

## 1. 문서 목적
본 문서는 우리은행 기업여신 AI 시연 화면의 컬러 정책과 단말 UI 원칙을 정의한다.
본 정책은 우리은행 공식 웹 채널과 로고 자산에서 확인되는 시각 요소를 바탕으로, 시연 환경에 맞게 해석한 구현 기준이다.

## 2. 기준 출처
- 우리은행 공식 홈페이지 계열 채널
- 우리은행 로고 자산
- `renewal/assets/img/h1_01.png`

## 3. 해석 원칙
- 기본 브랜드 축은 `우리 블루`와 `보조 골드`로 본다.
- 실제 업무 단말은 브랜드색을 전면에 쓰기보다, `연한 회청색 + 중간 회청색 + 브랜드 블루 포인트` 구조가 적합하다.
- 프로젝터 시연 환경에서는 채도가 높은 색보다 명도 차이가 충분한 색 조합이 더 중요하다.

## 5. 화면유형별 컬러 정책
## 5.1 단말 화면
- 목표: 실제 우리은행 영업점 업무 단말처럼 보이게 한다.
- 기본 배경은 회청색 계열을 사용한다.
- 브랜드 블루는 버튼, 상태 강조, 선택 상태에만 제한적으로 사용한다.
- 상단 크롬은 `흰색~회청색` 그라데이션과 얇은 경계선 중심으로 구성한다.
- 과도한 블루 배경 카드, 대형 히어로, 강한 네온톤은 금지한다.

## 5.2 Agent 화면
- 목표: 백그라운드 처리 모니터처럼 보이게 한다.
- 단말보다 더 어두운 네이비/차콜 계열 헤더나 배너 사용 가능
- 처리 상태와 시스템성 강조를 위해 대비를 높인다.
- 다만 경고/보완 포인트는 단말과 동일한 상태색 체계를 사용한다.

## 5.3 브릿지 화면
- 목표: 사람의 검토와 통제가 중심인 화면으로 보이게 한다.
- 흰색 패널과 중립 배경을 유지한다.
- 강조색은 브랜드 블루보다 `경고 골드`, `검토용 네이비`, `오류 레드` 우선
- 근거 문서 하이라이트와 승인/보완 액션의 구분이 명확해야 한다.

## 5.4 브릿지 미니 화면
- 목표: 짧은 시간에 핵심만 이해시키는 알림
- 브랜드 블루 또는 경고 골드를 포인트로 사용하되, 정보량은 최소화한다.

## 6. 단말 UI 원칙
- 단말에는 실제로 표시될 법한 업무 요소만 남긴다.
- 설명형 카드, 시나리오용 카피, AI 제언은 단말에서 제거한다.
- 단말 상단에는 우리은행 로고와 같은 톤의 `WINI` 명칭, 사용자/영업점 정보, 단말번호가 표시된다.
- 단말 본문은 `입력 폼`, `조회 결과 표`, `등록 현황`, `업무 메시지`, `액션 버튼` 중심으로 구성한다.
- 단말의 설명 문구는 최소화하고, 실제 단말에 존재할 법한 안내나 조회 결과만 남긴다.

## 7. 시인성 원칙
- 프로젝터 환경과 원거리 관람을 고려해 본문 기본 글자 크기를 기존 웹보다 크게 설정한다.
- 화면에 출력되는 텍스트의 최소 크기는 `15px`로 보되, 실제 시연 화면의 기본값은 그보다 충분히 크게 잡는다.
- 권장 최소 기준
- 타이틀: `44~48px`
- 탭/버튼: `18~20px`
- 패널 제목: `26~28px`
- 표/입력값: `17~18px`
- 상태 수치: `30px` 내외
- 정보는 한 화면 안에 닫되, 작은 보조 문구보다 핵심 수치와 상태가 먼저 읽히도록 한다.

## 8. 구현 적용 원칙
- 단말 화면에는 `우리은행 + WINI` 상단 크롬을 공통 적용한다.
- G1, G2, G7, G8은 같은 단말 크롬과 컬러 정책을 따른다.
- Agent, 브릿지, 브릿지 미니는 같은 상태색 체계를 공유하되 배경과 톤을 분리한다.

## 9. 결론
향후 단말 구현은 "브랜드를 입힌 예쁜 웹 화면"이 아니라 "우리은행 업무 단말을 시연용으로 정제한 화면"이어야 한다.
컬러는 브랜드를 암시하되 절제해서 사용하고, 화면 구조는 실제 단말의 업무성, 시인성, 안정성을 우선해야 한다.
