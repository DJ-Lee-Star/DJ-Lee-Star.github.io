# 기업여신 AI 시연 화면설계 방향성 가이드

## 1. 문서 목적
본 문서는 우리은행 기업여신 AI 시연 화면의 공통 디자인 방향과 레이아웃 기준을 정의한다.
이번 개정의 핵심은 `1920 x 1080` 해상도와 `단말 -> Agent -> 브릿지 -> 단말` 업무 흐름을 시각적으로 명확히 드러내는 것이다.

---

## 2. 공통 기준

### 2.1 해상도 기준
- 모든 주요 화면은 `1920 x 1080` 고정 캔버스를 사용한다.
- 발표용 디스플레이와 브라우저 전체화면 기준에서 잘림 없이 보여야 한다.
- 스크롤 없이 한 화면 안에 메시지가 닫히도록 설계한다.

### 2.2 시연 안정성 기준
- 동적 로딩, 팝업 파일 선택, 외부 연동 대기는 시연 화면에서 직접 노출하지 않는다.
- 시연 중 사용자는 최소 클릭만 수행하도록 한다.
- 버튼, 모달, 알림은 중복 실행을 방지하는 구조로 설계한다.
- 모든 주요 데이터는 고정 시연 데이터 기반으로 렌더링한다.

### 2.3 브랜드 기준
- 고객사는 우리은행이다.
- 화면은 은행 업무 시스템의 신뢰감과 단정함을 우선한다.
- 과장된 AI 랜딩 화면 톤은 지양한다.
- 우리다움 폰트는 제목, 숫자, 상태 라벨, 주요 섹션 헤더 중심으로 적극 활용한다.

---

## 3. 화면군별 방향

## 3.1 단말 화면
### 역할
- 신청 정보 입력
- 서류 등록
- AI 분석 요청
- 결과 반영 확인
- 심사/보완 분기

### 디자인 원칙
- 은행 영업점 업무 단말처럼 보여야 한다.
- 소개형 히어로보다 업무 툴바, 탭 바, 상태 스트립, 입력 패널, 결과 패널 구조가 적합하다.
- 상단은 업무 컨텍스트, 중앙은 입력/조회, 하단은 액션으로 구분한다.
- 카드 전시형 UI보다 표, 폼, 상태바, 업무 메시지 박스 위주의 평평한 레이아웃을 우선한다.
- 우리은행 영업점 단말처럼 차분한 회색/청색 베이스, 얇은 경계선, 촘촘한 정보 밀도를 사용한다.
- 단말에는 AI 제언, AI 판단, AI 추천 문구를 노출하지 않는다.
- 제출서류 목록은 AI 산출 결과가 아니라 상품/업무 기준 안내 또는 등록 현황으로만 표현한다.

### 레이아웃 권장안
- 상단: 우리은행 로고, `WINI` 명칭, 업무명, 사용자/영업점 정보
- 상단 보조줄: 업무바, 신청번호, 담당자, 단계 정보
- 중단: 탭 바와 상태 스트립
- 본문: 좌측 주업무 영역 / 우측 요약 및 결과 영역
- 하단: 액션 바

### 추가 원칙
- 단말 본문에는 실제 단말에 표시될 법한 요소만 남긴다.
- 긴 설명 박스 대신 조회 결과 표, 등록 현황 표, 짧은 업무 메시지 스트립을 사용한다.
- 프로젝터 시연 기준으로 표/입력값도 충분히 크게 설계한다.

### 금지 사항
- 단말에서 AI 기능을 직접 조작하는 표현
- 단말에서 AI 제언을 직접 확인하는 표현
- 단말에서 AI가 서류를 판단하거나 산출하는 표현
- 과도한 히어로 배너
- 랜딩페이지형 큰 카피 중심 구성

---

## 3.2 Agent 화면
### 역할
- 요청된 건의 자동 처리 현황 표현
- OCR, 검증, 제언 생성 결과 표시
- 실제 사용자가 조작하는 화면이 아니라 내부 처리 상태를 시연용으로 관찰하는 모니터 화면

### 디자인 원칙
- 별도 처리 엔진 또는 운영 모니터처럼 보여야 한다.
- 사용자가 조작하는 느낌을 제거하고, 자동 처리 로그와 상태 감시 느낌을 우선한다.
- "실제 화면 없이 백그라운드에서 실행된다"는 메시지가 헤더와 본문에 모두 드러나야 한다.
- 처리 큐, 진행률, 이상 징후, 브릿지 전달 사유를 명확히 보여 준다.

### 레이아웃 권장안
- 상단: 내부 처리 배너와 요청 출처 정보
- 중단: 단계 흐름 또는 처리 스트립
- 본문: 좌측 처리 큐 / 중앙 결과 로그 / 우측 브릿지 전달 요약

---

## 3.3 브릿지 화면
### 역할
- 근거 확인
- 수정/승인/보완 판단
- 단말 반영 승인

### 디자인 원칙
- 사람의 판단과 책임이 존재하는 통제 화면이어야 한다.
- 좌측 목록, 중앙 근거, 우측 판단 액션 구조가 가장 명확하다.
- AI 결과보다 근거와 반영 결정이 더 중요하게 보여야 한다.

### 레이아웃 권장안
- 좌측: 검토 대상 목록
- 중앙: 문서 근거 및 하이라이트
- 우측: 검토 값, 수정 값, 승인/보완 액션

---

## 3.4 브릿지 미니 화면
### 역할
- 브릿지 검토 요청 또는 처리 완료 알림

### 디자인 원칙
- 전체 화면이 아니라 오버레이형 카드가 적합하다.
- 3초 안에 이해되는 정보량으로 제한한다.
- 신청번호, 핵심 사유, 바로가기 액션만 남긴다.

---

## 4. 폰트 활용 방향
- `WooridaumB`: 메인 타이틀, 카드 수치, 상태 라벨, 섹션 헤더
- `WooridaumR`: 본문, 입력값, 설명 문구
- `WooridaumL`: 보조 설명, 메타 정보

### 적용 원칙
- 모든 텍스트를 우리다움으로 통일하기보다, 중요한 정보 위계에 집중해 사용한다.
- 숫자, 신청번호, 금액, 상태값은 가독성을 최우선으로 한다.
- OCR 원문, 긴 테이블 값, 메타 데이터는 필요 시 보조 폰트 혼용이 가능하다.

---

## 5. 이번 개정에서 반드시 반영할 메시지
- 단말은 AI 직접 사용 화면이 아니다.
- 단말은 요청을 보내고, 결과를 받는 화면이다.
- Agent는 자동 처리 채널이다.
- 브릿지는 검토와 통제 채널이다.
- 최종 결과는 다시 단말 업무 화면에 반영된다.

---

## 6. 결론
향후 상세 설계와 구현은 단순히 예쁜 화면이 아니라, 우리은행 영업점/심사 업무 흐름을 설득력 있게 보여 주는 구조여야 한다.
시연 화면은 다음 순서를 시각적으로 분명히 드러내야 한다.

`단말 입력/등록 -> AI 분석 요청 -> Agent 자동 처리 -> 브릿지 검토 -> 단말 반영`
