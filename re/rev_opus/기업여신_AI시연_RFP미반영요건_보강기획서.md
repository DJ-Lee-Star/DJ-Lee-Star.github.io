# 기업여신 AI Agent 시연 — RFP 미반영 요건 보강 기획서

> 본 문서는 RFP 요건 전수를 현행 시연 대비 반영 현황으로 분류하고,  
> **미반영·부분반영 요건의 시연 보강 전략**을 수립한다.  
> "시연 설명회에서 평가위원이 이 요건을 질문했을 때,  
> 화면으로 보여줄 수 있는가?" 를 최종 판단 기준으로 삼는다.

---

## 1. RFP 요건 전수 분류

### 1.1 분류 기준

| 등급 | 정의 | 시연 관점 |
|------|------|----------|
| **반영** | 현행 시연에서 E2E 중 해당 요건을 화면으로 보여주고 있음 | 질문 시 화면 캡처로 답변 가능 |
| **부분반영** | UI는 있으나 핵심 동작 누락, 또는 시뮬레이션 수준 | 설명은 가능하나 "실제 동작"을 보여주기 어려움 |
| **미반영** | 현행 시연에 해당 화면·기능이 전혀 없음 | 질문 시 "향후 구현 예정"으로만 답변 가능 (경쟁 열위) |

| 등급 | 건수 | 비율 |

| **반영** | 6건 | 15% |
| **부분반영** | 12건 | 30% |
| **미반영** | 22건 | **55%** |

> **기업부동산담보대출 시나리오 변경으로 담보서류 관련 2건이 부분반영으로 개선.**

> 평가위원이 해당 요건을 질문하면 "향후 구현 예정"으로만 답변해야 하며, 이는 경쟁 제안 대비 명확한 약점.

## 3. 보강 전략

### 3.1 전략 원칙

| 원칙 | 설명 |
|------|------|
| **시뮬레이션 우선** | 백엔드 없이 하드코딩 데이터 + 타이핑 애니메이션으로 "동작하는 것처럼" 보이게 함. 실제 AI/API 연동은 2차 과제 |
| **기존 화면 확장** | 신규 화면 최소화. 기존 P1~P12에 요건을 "녹여 넣어" 추가 시연 시간을 최소화 |
| **질문 방어형** | "이 요건은요?" 라는 질문에 "이 화면에서 보시면..." 으로 즉시 답변 가능한 상태 목표 |
| **핵심 집중** | 40건 전부 대응이 아니라, 평가위원이 반드시 물어볼 7~10건을 완벽히 대응 |

### 3.2 보강 유형

| 유형 | 방법 | 대상 요건 수 |
|------|------|------------|
| **A. 신규 화면** | 새 HTML 페이지 추가 | 2건 (P0, P13) |
| **B. 기존 화면 확장** | 기존 화면에 컴포넌트/영역 추가 | 8건 |
| **C. 데이터 보강** | 데이터 모듈에 필드/시나리오 추가 | 3건 |
| **D. 답변 자료** | 시연에 보여주진 않지만, 질문 시 설명할 근거 문서 | 5건 |
| **E. 명시적 유보** | 시연 범위 외 (사후관리, 백엔드 등) | 6건 |
